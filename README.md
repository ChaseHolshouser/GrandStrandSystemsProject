# GrandStrandSystemsProject
Final project for my CS-320: Software Quality Assurance class

How can I ensure that my code, program, or software is functional and secure?
I can ensure my software is functional and secure by following coding best practices, validating user input, and writing thorough unit tests to verify that each feature works as intended. Regular code reviews, debugging, and testing edge cases also help identify defects and vulnerabilities before the software is released.

How do I interpret user needs and incorporate them into a program?
I begin by carefully reviewing the project requirements to understand what the user expects the software to do. From there, I translate those requirements into specific features and use testing to verify that the program behaves according to the user's needs.

How do I approach designing software?
I approach software design by first breaking the project into smaller, manageable components that each have a specific responsibility. Once the overall structure is planned, I implement each component individually, test it thoroughly, and then integrate the pieces together to create a complete and reliable application.
